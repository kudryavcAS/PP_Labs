# Named Pipes Lab (Лабораторная работа №5)

Проект демонстрирует межпроцессное взаимодействие (IPC) в Windows с использованием именованных каналов (Named Pipes). Реализована модель "Клиент-Сервер" с синхронизацией доступа к файлу (Reader-Writer Lock).

## Требования

*   **ОС:** Windows 10/11 (используется WinAPI).
*   **IDE:** Visual Studio 2019 или 2022.
*   **Компоненты VS:** "Разработка классических приложений на C++" (Desktop development with C++).

## Структура проекта

*   **Server**: Управляет базой данных сотрудников, слушает подключения, синхронизирует потоки.
*   **Client**: Консольное приложение для чтения и модификации записей.
*   **ServerTests**: Юнит-тесты для проверки логики сервера (Microsoft Native Unit Test Framework).

## Как запустить

1.  Откройте файл `Lab 5.sln` в Visual Studio.
2.  В обозревателе решений нажмите правой кнопкой на проект **Server** -> **Set as Startup Project** (Назначить запускаемым).
3.  Нажмите **F5** (или кнопку Run).
4.  Сервер запустится и предложит создать файл базы данных. Введите имя файла (например, `db.bin`), количество сотрудников и данные.
5.  Сервер автоматически запустит указанное число клиентов.

## Как запустить тесты

1.  В Visual Studio откройте меню **Test** -> **Test Explorer** (Обозреватель тестов).
2.  Дождитесь, пока VS обнаружит тесты.
3.  Нажмите кнопку **Run All** (Запустить все).

## Особенности реализации

*   Используется **WinAPI** (`CreateNamedPipe`, `CreateFile`, `LockFileEx`, `CreateThread`).
*   Синхронизация доступа: **Exclusive Lock** для записи, **Shared Lock** для чтения.
*   Кодировка проекта: **Multi-Byte Character Set**.
